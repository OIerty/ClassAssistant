# 端口调用示例

这份示例文件会随发布包一起打进 zip，方便你快速验证不同端口。

## 开发默认端口

- 后端健康检查: http://127.0.0.1:8765/api/health
- 设置读取: http://127.0.0.1:8765/api/settings
- 资料列表: http://127.0.0.1:8765/api/cite_files
- Swagger: http://127.0.0.1:8765/docs

## 打包验证端口

打包脚本会用 18765 做临时健康检查，避免和开发环境冲突。

- 后端健康检查: http://127.0.0.1:18765/api/health
- 启动监控示例: POST http://127.0.0.1:18765/api/start_monitor
- 停止监控示例: POST http://127.0.0.1:18765/api/stop_monitor

## 前端调用示例

前端默认通过 `http://127.0.0.1:8765/api` 访问后端。
如果你修改了 `API_PORT`，前端设置页和前端源码里的接口基址需要保持一致。

示例:

- `http://127.0.0.1:8765/api/upload_ppt`
- `http://127.0.0.1:8765/api/start_monitor`
- `http://127.0.0.1:8765/api/ingest_asr_text`

## 其他端口说明

- 8765: 日常开发默认端口
- 18765: 打包校验专用端口

如果你把后端改到别的端口，所有调用地址都要同步修改。
